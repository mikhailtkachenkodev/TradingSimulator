# Техническое задание

Написать программу на **C++** с использованием **только стандартной библиотеки**, которая, используя модель
*геометрического броуновского движения*, моделирует эволюцию цены токена во времени.

Каждое обновление цены должно пересчитывать два скользящих средних, используемых для обнаружения сигналов на вход в
позицию и на выход из неё по токену (long / short при разворотах цены).

### Интерфейс взаимодействия

Сигнал должен вызывать:

```cpp
auto OrderManager::SendOrder(const Order &order) -> OrderIdentifier
```

который должен сохранять информацию по ордеру, симулируя его отправку.

Ответом на отправку ордера является callback:

```cpp
auto IHandler::HandleRequestReply(const OrderIdentifier id, const Status reply_status, const std::string_view reply_error) -> void
```

реализуемый в классе, который собирает и отправляет ордер.

* `OrderIdentifier` — уникальный идентификатор ордера.
* `Status` — перечисление (enum) возможных исходов.

В ряде случаев, случайным образом, ответ должен возвращать `Status::Rejected` (другие варианты — на усмотрение
реализующего).

### Входные параметры

1. Начальное значение цены.
2. Среднее значение тренда цены.
3. Волатильность цены.
4. Минимальное и максимальное время между предыдущей и следующей сгенерированной ценой.
5. Минимальный и максимальный объём для каждого значения цены.
6. Минимальное и максимальное значение позиции.
7. Процент отклонённых ордеров (процент rejected).

### Выходные данные

1. Файл с эволюцией цены во времени.
2. Файл с отправленными ордерами (сторона, цена, объём, статус (reply status), текст ошибки (reply error), текущий PnL).

### Требования к реализации

По возможности **избегать наследования** (кроме реализации интерфейса callback `IHandler`).
